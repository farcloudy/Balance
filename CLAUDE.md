## 项目概述

Balance 是一个设计为在 Windows 本地运行的个人记账系统。它实现了复式记账法和权责发生制会计原则，并包含 OCR 功能，用于处理银行回单和支付截图。

## 技术栈

**后端** (位于 `Backend/` 目录):
- Node.js + MongoDB
- 在 Windows 本地运行

**前端** (位于 `Fronted/` 目录 - 注意目录名):
- Vue 3.5.25 + Vite 7.3.1
- Pinia 3.0.4 状态管理
- Vue Router 5.0.3
- Axios 请求库
- 已完成初始化配置

## 核心功能

1. **身份验证**: 基础登录系统，防止未授权访问
2. **视觉模型集成**: 使用视觉语言模型（如 Claude Vision、GPT-4 Vision 等）直接处理银行回单、电子支付截图和聊天记录图片，一步完成信息提取和会计科目建议
3. **账套管理**:
   - 每个用户可创建多个独立账套
   - 权责发生制会计
   - 生成报表：月度、季度、半年度和年度
4. **AI API 接口**: 统一的 AI API 接口，用于视觉识别和智能建议
5. **对象存储**: 参考实现位于 `D:\Work\Sheet\backend\routers`

## 重要技术决策

### 数值精度
所有货币金额必须以整数形式存储在数据库中（乘以 100）。前端显示时保留 2 位小数。这可以防止浮点数精度错误。

**示例**:
- 用户输入: ¥123.45
- 数据库存储: 12345 (整数)
- 前端显示: 123.45

### 代码风格
- 使用 4 个空格缩进（不使用制表符）
- 单引号、不使用分号
- 已配置 Prettier 和 ESLint
- 适用于前端和后端代码

### 多租户
每个用户可以创建和管理多个独立的账套。确保用户之间以及用户的各个账套之间的数据隔离。

## 架构说明

### 数据模型考虑
- 实现标准的复式记账（借贷系统）
- 支持权责发生制会计
- 会计科目层级结构
- 交易日记账，包含完整的审计追踪

### 视觉模型工作流程
使用视觉语言模型（VLM）直接处理图片，一步完成信息提取和智能建议：

**输入**: 银行回单/支付截图/聊天记录图片

**输出**:
- 支付账户/方式
- 金额
- 日期/时间
- 收款方/商户
- 建议的会计科目
- 建议的交易类型
- 其他相关信息

**技术方案**:
- 复用统一的 AI API 接口（支持 Claude Vision、GPT-4 Vision 等）
- 无需单独的 OCR 服务
- 视觉模型可以理解图片上下文，提供更准确的会计科目建议
